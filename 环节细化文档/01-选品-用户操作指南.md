# ① 选品环节 — 用户输入什么 → 完整流程 → 产出什么

> 用户视角说明书。客观回答："我作为商家，输入什么，会发生什么，最后拿到什么"

---

## 一、用户输入（两种方式）

### 方式 A：自然语言（最简单）
直接说一句中文：

```
"我想做蓝牙耳机这个品类的选品调研。"
"分析一下美区智能手表市场，预算月度 5 万美元，做自有品牌精品。"
"我想做 TikTok Shop 的厨房小工具，单 SKU MOQ 1000 内能接受的。"
```

Agent 会反问 6 项澄清。

### 方式 B：结构化参数（API 调用）
直接把 6 项需求一次填好：
```json
POST http://127.0.0.1:8001/chat
{
  "text": "蓝牙耳机选品调研",
  "params": {
    "market": "US",
    "platforms": ["amazon", "walmart"],
    "budget_monthly_usd": 50000,
    "moq_max": 1000,
    "logistics": "FBA",
    "positioning": "自有品牌中端精品",
    "exclude": ["Apple", "Sony 专利雷区"]
  }
}
```

## 二、可指定的平台（多选）

**当前实测可抓的 6 个平台**（你可以单选/多选）：

| 平台 | 状态 | 代理需求 | 备注 |
|---|---|---|---|
| **Amazon** | ✅ 可抓 | 美国 IP（已配 xray） | 数据最全，BSR + 评论 + AI 摘要 |
| **Walmart** | ✅ 可抓 | 美国 IP | botasaurus 攻破 |
| **Temu** | ✅ 可抓 | 任意 | SPA，patchright 渲染 |
| **Shopee** | ✅ 可抓 | 任意 | 默认新加坡站，可换 .com.my/.tw |
| **AliExpress** | ✅ 可抓 | 美国 IP | botasaurus 过阿里 NC |
| **BestBuy** | ✅ 可抓 | 美国 IP | 反爬已突破 |

**当前不可抓的 2 个平台**（开源攻不下，需要付费）：

| 平台 | 障碍 | 解决方案 |
|---|---|---|
| **eBay** | 商业级反爬 | 需要 BrightData / Oxylabs 住宅代理 |
| **Etsy** | DataDome 商业反爬 | 需要 CapSolver 等付费打码服务 |

**多选支持**：Agent 收到 `platforms: ["amazon", "walmart", "shopee"]` 会循环每个平台抓一遍，结果汇总。

## 三、完整流程图（按 8 阶段方法论）

```
                      [用户输入]
                          │
                          ▼
┌─ 阶段 0：需求澄清 ──────────────────────────────────┐
│ Agent 反问：市场? 平台? 预算? 物流? 定位? 排除项?     │
│ 工具：load_skill                                      │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌─ 阶段 1：品类宏观（趋势洞察）→ 对应清单 [✓ 趋势洞察] ─┐
│ • get_trend(关键词, 地区) → Google Trends 12 月曲线   │
│ • discover_bsr_url + get_bestsellers_by_url           │
│   → Amazon BSR Top 50（真 ASIN/价格/评分/评论数）     │
│ • get_movers_shakers_by_url（24h 上升榜）             │
│ 自动入 ASIN 池                                         │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌─ 阶段 2：竞争格局（竞品分析-价格/销量/评分） ─────────┐
│ • search_products(平台, 关键词) — 多平台循环抓        │
│ • analyze_market_structure(products)                  │
│   → 价格分布 / 评分中位 / 品牌集中度 CR4              │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌─ 阶段 3：痛点挖掘（竞品分析-评论/卖点） ──────────────┐
│ • get_reviews_batch(asins=[5-10个真ASIN])             │
│   → 累计 40-130 条真评论 + 详情页 AI 摘要 + 关键词云  │
│ • analyze_reviews(reviews) — LLM 提炼 Top 5 痛点       │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌─ 阶段 4：候选品筛选 ─────────────────────────────────┐
│ • get_asin_pool() 看池子                              │
│ • LLM 从池中提名 3-5 个候选 ASIN                      │
│ • validate_candidate(asin) 强制校验在池中             │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌─ 阶段 5：利润测算 → 对应清单 [✓ 利润测算] ────────────┐
│ • get_real_procurement_cost(中文关键词) — 1688 真抓   │
│   失败时返回 error，引导走 search_1688 反查           │
│ • full_cost_breakdown(asin, sale_price, cost,         │
│     category, weight_oz, longest_in)                  │
│   → 14 项成本（采购+头程+关税+检测+FBA+佣金+广告...）  │
│   → 真实数据：USITC HTS 关税 / Amazon 2026 费率 /     │
│      实时汇率 / FBA Standard Size 费表                │
│ • stress_test(...) ACOS 翻倍 / 退货 15% / 汇率冲击     │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌─ 阶段 6：差异化定义 → 对应清单 [✓ 差异化机会点] ──────┐
│ • LLM 综合数据，给产品定义                            │
│   - 1-2 个核心改进点（来自真痛点）                    │
│   - 参数表 / 包装 / 定价节奏 / 品牌话术               │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌─ 阶段 7：风险扫描（蓝海/红海判断） ──────────────────┐
│ • quick_ip_check(关键词, 候选品牌名)                  │
│   → Google Patents + USPTO 商标检索                   │
│ • Agent 综合产品定义判断蓝海 vs 红海                  │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌─ 阶段 8：决策输出 → 对应清单 [✓ 选品决策] ────────────┐
│ • 模型自动切到 deepseek-v4-pro（深度推理）            │
│ • capture_evidence(asin) 给每个候选品截图存证         │
│ • 生成最终 markdown 报告 + JSON 数据                  │
└──────────────────────┬───────────────────────────────┘
                       ▼
                  [最终产出文件]
```

## 四、最终产出（你能拿到这些文件）

### 1. 决策报告 markdown
路径：`poc/p01_selection/reports/agent_final_<时间戳>.md`

包含 5 大块：
- **候选品决策表**：ASIN / 标题 / 售价 / 评分 / 评论数 / BSR / 月销区间 / 上架决策
- **每行带证据链接**：Amazon 详情页 URL + 本地截图链接
- **14 项成本测算**：每个候选品的完整成本拆解 + 数据来源标注（HTS/FBA/佣金/汇率）
- **风险清单**：专利、商标、政策、退货、汇率压测
- **90 天行动计划**：备货 / 测款 / 优化分期

### 2. 证据截图
路径：`poc/p01_selection/reports/evidence/<asin>_dp.png`

每个候选品自动截 2 张：
- Amazon 详情页快照
- Amazon 搜索结果页快照（含价格、评分对比）

### 3. 完整工具调用日志
路径：`poc/p01_selection/reports/agent_run_<时间戳>.log`

每一步带时间戳 + 工具名 + 参数 + 返回预览，可追溯。

### 4. 持久化数据
SQLite 数据库：`poc/p01_selection/data/agent.sqlite`
- `threads` 表：会话状态、累计 token、累计费用
- `messages` 表：所有用户/Agent 消息（JSONB parts 格式，对齐 backend steering）

### 5. SSE 流式数据（可选）
若用 backend API 调用：浏览器/客户端可以**实时**接收 Agent 的思考过程、工具调用、文本流式输出。

## 五、5 个子任务到流程的精确映射

回答你的核心问题"流程对应清单的哪一步"：

| 清单子任务 | 对应阶段 | 用了哪些工具 |
|---|---|---|
| ✅ **趋势洞察** (品类热度/季节性/关键词搜索量) | 阶段 1 | `get_trend`（pytrends） |
| ✅ **竞品分析** (价格/销量/评分/评论/卖点) | 阶段 2-3 | `search_products`、`get_bestsellers_by_url`、`get_reviews_batch`、`analyze_reviews` |
| ✅ **利润测算** (采购价/头程/佣金/广告/退货率) | 阶段 5 | `get_real_procurement_cost`（1688）、`full_cost_breakdown`（14 项）、`stress_test` |
| ✅ **蓝海/红海判断** + 差异化机会点 | 阶段 2 + 阶段 7 | `analyze_market_structure`（CR4）、`analyze_reviews`（痛点）、`quick_ip_check` |
| ✅ **选品决策** (是否上架/定价区间) | 阶段 8 | DeepSeek v4-pro 综合所有数据 + `capture_evidence` 截图 |

**5 个子任务都被工具链覆盖。**

## 六、实操示例（你现在就能跑）

### 启动服务
```powershell
# 1. Redis（已下载）
.\bin\redis\Redis-7.4.1-Windows-x64-msys2\redis-server.exe

# 2. xray 美国代理
.\.venv\Scripts\python.exe poc\p01_selection\proxy\setup_us_proxy.py

# 3. FastAPI 后端
.\.venv\Scripts\python.exe -m uvicorn poc.p01_selection.backend.app:app --port 8001
```

### 发起调研（命令行）
```powershell
.\.venv\Scripts\python.exe -c "import httpx; r=httpx.post('http://127.0.0.1:8001/chat', json={'text':'蓝牙耳机选品调研，美国 Amazon FBA，月度 5 万预算，自有品牌精品'}); print(r.json())"
```

返回：`{"thread_id": "...", "stream_id": "...", "status": "queued"}`

### 实时看流式输出
```powershell
.\.venv\Scripts\python.exe poc\p01_selection\probes\test_backend_e2e.py
```

### 查最终结果
```powershell
.\.venv\Scripts\python.exe -c "import httpx,json; r=httpx.get('http://127.0.0.1:8001/thread/<thread_id>'); print(json.dumps(r.json(), ensure_ascii=False, indent=2))"
```

## 七、什么还做不了（诚实交代）

| 能力 | 状态 | 原因 |
|---|---|---|
| eBay / Etsy 抓取 | ❌ | 商业反爬，需付费代理或打码服务 |
| 1688 自动比价 | ⚠️ | 阿里 NC 验证码挡住，已加 fallback 但不准 |
| 精确销量 | ⚠️ | Amazon 不公开，全行业都是 BSR 区间估算 |
| 精确搜索量 | ⚠️ | Google Ads API 需 API key + 付费 |
| 多语言 Listing 检查 | ❌ | 还没接入（属于环节 ④ 本地化） |
| 前端 UI | ❌ | SSE 客户端只有 Python 测试脚本，没有 React 页面 |
