# ① 选品环节 — 完整流程与驱动机制

> 回答用户提问：完整流程是什么？整体由什么驱动？中间过程长什么样？所有 vendor 项目如何参与？

---

## 一、整体由什么驱动？

**双驱动**：
1. **业务驱动**：商家用自然语言提需求 → DeepSeek LLM 理解 → Agent 决策下一步
2. **方法论驱动**：`skills/procurement-research.md`（8 阶段方法论 markdown）→ Agent 第一步加载，按阶段推进，避免凭感觉走

LLM 不是简单回答，而是**调度工具采集真实数据 → 基于数据推理 → 给结论**。

## 三、中间过程详解（每一步发生了什么）

### Step 1（Flash）：加载方法论
```
🔧 [load_skill]({"skill_name": "procurement-research"})
   ↳ 0ms 返回 markdown 全文（8 阶段说明）
```

### Step 2（Flash）：阶段 0 反问需求
```
💭 [Agent]
请确认 6 项：
  1. 目标市场（US/EU/SEA/全球）
  2. 目标平台（Amazon/Walmart/独立站）
  3. 资金预算（月度采购+单 SKU MOQ）
  4. 物流方式（FBA/海外仓/自发货）
  5. 商家定位（白牌/自有品牌/代工）
  6. 排除项（敏感品类/品牌雷区）
📨 [自动答复]
  美国 + Amazon FBA + 月 5 万美元 / MOQ 500-1000 + FBA + 自有品牌中端 + 避开 Apple/Sony
```

### Step 3-5（Flash）：阶段 1 品类宏观
```
🔧 [get_trend]({"keyword": "wireless earbuds", "geo": "US"})
   ↳ {"early_avg": 6.3, "late_avg": 50.7, "direction": "上升", ...}
🔧 [get_trend]({"keyword": "bluetooth headphones", "geo": "US"})
🔧 [get_bestsellers]({"category": "electronics", "limit": 30})
   ↳ 30 件真实商品（Apple AirPods 4 / AirTag / Fire TV Stick / ...）
🔧 [get_movers_shakers]({"category": "electronics"})
   ↳ 24h 上升最快榜
```

### Step 6-9（Flash）：阶段 2-3 竞争格局 + 痛点
```
🔧 [search_products]({"platform": "amazon", "keyword": "wireless earbuds bluetooth"})
🔧 [search_products]({"platform": "walmart", ...})        ← 美国代理 + botasaurus 突破
🔧 [analyze_market_structure]({"products": [...]})
   ↳ 价格分布 / 评分中位 / 品牌 CR4
🔧 [get_reviews_batch]({"asins": [5-10 个竞品 ASIN]})
   ↳ 累计 40-80 条真实评论 + 关键词云
🔧 [analyze_reviews]({"reviews": [...]})
   ↳ LLM 提炼 5 大痛点 + 3 大卖点 + 差异化机会
```

### Step 10-11（Flash）：阶段 4 候选筛选
```
💭 [Agent]
基于上述数据，提名 3 个候选 SKU：
  A. 低价走量 $25.99
  B. 中端精品 $59.99
  C. 运动防汗 $69.99
```

### Step 12-14（Pro）：阶段 5-8 决策（深度推理）
```
━━━ 模型切换 deepseek-v4-pro ━━━
🔧 [full_cost_breakdown]({"sale_price": 25.99, "procurement_cost": 10.4, ...}) × 3
   ↳ 14 项成本拆解 + 净利 + 毛利率 + 盈亏点
🔧 [stress_test] × 2
   ↳ ACOS 翻倍 / 退货 15% / 汇率 -10% 时还能否盈利
🔧 [quick_ip_check]({"keyword": "wireless earbuds", "brand_candidate": "Earis"})
   ↳ Google Patents + USPTO 商标检索

💭 [Agent — 最终报告]
| SKU | 售价 | 净利 | 毛利率 | 盈亏点 | 决策 |
| ---- | ---- | ---- | ----- | ----- | ---- |
| A    | $25.99 | $2.11 | 8.1%  | -      | ❌ 放弃 |
| B    | $59.99 | $9.83 | 16.4% | 190/月 | ✅ 上架（主推） |
| C    | $69.99 | $11.48| 18.5% | 175/月 | ✅ 上架 |

主推：Earis ComfortFit @ $59.99
差异化：「小耳舒适 + 5 耳塞 + 鲨鱼鳍稳固」
风险清单 / 90 天计划 / 财务摘要
```

## 四、所有 vendor 项目的角色（无一闲置）

### 🔥 主力（每次都跑）
| 层 | 项目 | 用途 |
|---|---|---|
| L1 | `curl_cffi` | TLS 指纹伪装 HTTP（最快） |
| L2 | `Scrapling` | 自适应解析 + stealth |
| L3 | `patchright` | Playwright 反检测（SPA 站） |
| L6 | `botasaurus` | google_get + bypass_cloudflare（攻坚） |
| 趋势 | `pytrends` | Google Trends |
| 编排 | `tenacity` | 失败重试装饰器 |
| LLM | DeepSeek `v4-flash` / `v4-pro` | 模型切换 |

### 🛡 备胎（主力失败时自动接管）
| 层 | 项目 | 触发条件 |
|---|---|---|
| L4 | `pydoll` | patchright/botasaurus 都失败时 |
| L5 | `SeleniumBase` (UC Mode) | 上述都失败时的大杀器 |
| L7 | `camoufox` | Firefox 反指纹（浏览器内核下载后启用） |
| 验证码 | `FlareSolverr` (Docker 服务) | 遇到 Cloudflare/DDoS-GUARD 时启用 |
| 代理池 | `proxy_pool` (Docker 服务) | 当 xray 单代理失效时启用免费代理池 |
| 指纹 | `fingerprint-suite` | botasaurus 内部已用 |

### 🔧 增强工具（按需调用）
| 项目 | Agent 工具名 | 用途 |
|---|---|---|
| `crawl4ai` | `webpage_to_markdown` | 把对手 Listing/认证页转成干净 markdown 给 LLM 深度分析 |
| `markitdown` | `file_to_markdown` | 供应商 PDF 报价 / 专利 PDF / 认证文档 → markdown |
| `ScrapeGraphAI` | (在 extras.py 备用) | 页面结构变化大时用 LLM 直接抽字段 |
| `thefuzz` | (在 trends.py 备用) | 关键词模糊匹配/扩展 |

### 📅 调度（生产环境启用）
| 项目 | 用途 |
|---|---|
| `APScheduler` | 定时刷 BSR/价格快照（生产环境定时任务） |
| `Prefect` | 多环节合并时的 DAG 编排（暂时不需要） |

### 📊 看板（独立部署）
| 项目 | 部署方式 |
|---|---|
| `Superset` / `Metabase` | Docker 独立部署，连主库出选品看板 |

**结论：21 个 vendor 项目里 — 7 个主力、6 个备胎、4 个增强、2 个调度、2 个看板。**
**全部就位，按场景激活。**
