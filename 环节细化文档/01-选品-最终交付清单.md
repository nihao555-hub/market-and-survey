## 一、核心交付物（5 件套）

### 📄 1. 商家版报告（_merchant.md）— 主产出
**给商家：1 分钟决策 + 按需展开详情**
- 顶部 1 页：做不做 / 主推什么 / 预期利润 / 风险等级
- 中价值章节用 `<details>` 折叠（趋势/竞争/痛点原文/IP明细），按需展开
- 去掉开发者向水分（工具调用表、装饰线）
- 自动产出，无需手动编辑

### 📄 2. 一页摘要（_1page.md）— 极简版
**1 分钟读完的核心决策**：执行汇总表 + 风险等级 + 主推决策

### 📄 3. 5 页详情（_detail.md）— 给运营团队
含 5 大阶段核心结论 + 候选品对比表 + 数据来源声明

### 📄 4. 完整版（_full.md）— 留档/可溯源
全量数据 + 每个数据点的 source URL，给"采购决策可追溯"用

### 📄 5. PDF 版（.pdf）— 直接给商家发邮件
完整版 markdown 转 PDF（markdown-pdf，不需 GTK）

## 五、零编造保证（数据真实性）

1. **Stage 5 利润测算拿不到采购成本时** → 整章只输出"待用户提供清单"，绝不写假数字
2. **ACOS / 退货率无公开真实源** → 不当真值用，跑蒙特卡洛分布
3. **每个数据点带 source_url** → 报告可溯源
4. **traceability_check** → 自动校验报告里每个 ASIN 声明真实
5. **Keepa PNG 给 LLM 读时，用像素分析还原成趋势文本**（DeepSeek 无视觉，避免编造图里内容）
