# ① 选品 — 数据真实性矩阵（最终诚实版）

> 用户要求："这里都是真实数据呢" "不允许编造" "每个都是有证据来源的"
> 本文档对每一个 Agent 会用到的数据点逐一标注来源、真实性等级、证据 URL。

---

## 真实性等级定义

| 等级 | 含义 |
|---|---|
| 🟢 **真实抓取** | 实时从源站抓到，可附 URL 截图 |
| 🟡 **真实计算** | 基于真实抓取数据用公式算（如 BSR 排名是真，月销量是公式估算） |
| 🟠 **公开行业数据** | 来自公开费率表/官方 API（如 Amazon 2026 佣金费率） |
| 🔴 **估算/区间** | 公式或经验值，已明确标注 `real_data: false` |
| ❌ **暂不可用** | 接不到，明确返回 error |

## 🔴 仍需人工录入或付费 API 校准的部分

这 5 项当前默认值是行业经验，**应该让用户在 API 调用时覆盖**：

| 参数 | 当前默认 | 建议来源 |
|---|---|---|
| `shipping_per_unit_to_fba` | $4.5/件 | 真实货代报价（如 4PX/燕文/ESL） |
| `test_certification_amortized` | $0.30/件 | SGS/CTI/UL 报价 + MOQ 摊销 |
| `return_rate` | 8% | Amazon Seller Central Reports API |
| `ad_acos` | 20% | 商家自身历史店铺数据 |
| `payment_fee_rate` | 1.3% | PingPong/Airwallex 实际费率 |

**API 设计**：用户在 POST /chat 时可传 `cost_overrides` 参数覆盖默认值，让公式用真实店铺数据。

## 总结

**真实性分布**：
- 🟢 真实抓取：Trends、BSR、商品列表、评论、专利搜索、汇率
- 🟠 公开行业数据：关税、佣金、FBA Fee（Amazon/USITC 公开）
- 🔴 估算（明确标注）：月销量区间、5 个成本默认值
- ❌ 暂不可用：精确搜索量、1688 品类粗估、商标精确判定、eBay/Etsy 抓取

**没有"凭空编造"的数据**：
- 候选品必须从 ASIN 池选（强制校验）
- 利润测算的输入要求 LLM 调用真实工具拿（不允许直接传数字）
- 所有估算项都返回 `real_data: false` + warning
