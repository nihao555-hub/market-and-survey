# ① 选品 — 理想完整流程 vs 当前 PoC 流程（差距分析）

> 用户原话："比如我要调研市场中蓝牙耳机的，我们的理想完整流程是什么，和现在有什么偏差吗？"
> 这份文档拆出**资深跨境选品专员**真实做调研的完整方法论，再对比当前 Agent 流程，明确差距。

---

## 一、理想完整流程（资深选品专员怎么做）

### 阶段 0 · 需求澄清（Agent 第一步该做但没做）
和商家把需求搞清楚再行动。要问清：
- 目标市场（美国/欧洲/东南亚？影响平台、税、合规）
- 目标平台（Amazon / Walmart / 独立站？影响佣金、规则）
- 资金/MOQ 区间（决定打法是低价铺货还是中高端精品）
- 物流方式（FBA / 海外仓 / 自发货？决定成本结构）
- 商家定位（白牌 / 自有品牌 / 代工？影响差异化路径）
- 排除项（品牌、专利雷区、敏感品类）

> 当前 PoC：直接接受一句话就开干，没澄清就抓数据。导致后面利润测算都是凭空假设。

### 阶段 1 · 品类宏观（市场是不是值得做）
1. **Google Trends** — 看 5 年/12 月趋势，判上升/下降/季节性
2. **平台搜索量** — Amazon 搜索量 / 独立站 SEO 量
3. **市场规模** — 类目年 GMV、CAGR（公开数据/Statista 等）
4. **季节性日历** — 标出旺季月份，决定备货节奏
5. **政策/合规** — 欧美 CE/FCC 认证？平台限品？

> 当前 PoC：只做了 Google Trends 一项，单一信号定生死。

### 阶段 2 · 竞争格局（蓝海还是红海）
1. **Top 100 BSR / Best Sellers** — 该类目销量分布
2. **Movers & Shakers** — 24 小时上升最快的（开源金矿）
3. **品牌集中度（CR4/CR10）** — 头部 4/10 个品牌占多少份额
4. **价格带分布直方图** — 找空白价格区间
5. **新品入场速度** — 每月新增多少 SKU
6. **评分分布** — 整个类目的评分中位数（评分门槛）
7. **评论数 vs 上架时长** — 看流量获取难度

> 当前 PoC：抓了 16 个搜索结果，凭这判断"红海"过于粗糙。没看 BSR、没看品牌集中度、没看价格带空白。

### 阶段 3 · 痛点挖掘（差异化机会从哪来）
1. **从 1-3 星差评里挖痛点** — 用户最不爽什么
2. **聚类痛点频次** — 哪些抱怨重复出现
3. **从问答(Q&A)里挖未满足需求**
4. **看小红书/TikTok/Reddit 等社媒的吐槽**
5. **跨平台横评** — Amazon/Walmart 上同品类痛点是否一致

> 当前 PoC：只用了 10 条样例评论（不是真抓的），LLM 提炼出 5 个痛点是好的，但样本太小且非真实。

### 阶段 4 · 候选品筛选（具体抓哪几个 SKU）
1. **维度评分**：BSR 估算月销 × 评分 × 评论数 × 价格带 × 新旧度
2. **三维矩阵**：高需求 + 中竞争 + 高毛利 = 优选
3. **画像**：每个候选 SKU 的画像卡（标题/卖点/差评/价格/月销）
4. **对标分析**：选 3-5 个最强竞品深度拆解（A+ 页面、视频、定价节奏）

> 当前 PoC：没有这一步。直接从搜索结果选了几个就开始算利润。

### 阶段 5 · 利润可行性（钱算清楚）
1. **完整成本拆解**（采购+检测+头程+清关+关税+VAT+海外仓+末端配送+平台佣金+FBA仓储费+广告+退货+客服+汇率+融资）
2. **多价格场景测算**（低/中/高三档定价）
3. **盈亏平衡点**（每月最少卖多少件回本）
4. **现金流模型**（备货周期+回款周期，资金占用多少天）
5. **风险压测**（广告 ACOS 翻倍/退货率 15%/汇率波动 10% 时还能活吗）

> 当前 PoC：只算了 3 个价位的"售价-成本-费用=净利"，没含运费/关税/VAT/汇率/盈亏点/现金流。

### 阶段 6 · 差异化产品定义（你要做的版本是什么）
1. **核心差异化点**（基于痛点 → 1-2 个杀手级改进）
2. **技术参数表**（哪里超过竞品、哪里持平、哪里牺牲）
3. **包装/赠品/捆绑卖点**
4. **品牌故事/定位话术**
5. **目标定价 + 折扣节奏**

> 当前 PoC：直接给"不虚标续航+不断连"这两个差异化点，但没有完整的产品定义、参数表、定价节奏。

### 阶段 7 · 风险扫描（避雷）
1. **专利风险**（Google Patents / USPTO 检索）
2. **品牌侵权**（商标查询，避开"Apple/Sony"）
3. **平台政策风险**（限品、3C 认证、安全证书）
4. **物流风险**（电池类/液体/磁性的运输限制）
5. **侵权诉讼史**（某些品类是诉讼重灾区）

> 当前 PoC：完全没做这一步。专利和品牌侵权是跨境的高频死穴。

### 阶段 8 · 决策输出
- 候选 SKU 列表 + 每个的"上架/观察/放弃"+ 完整理由
- 推荐主推 1-3 款，给详细产品定义
- 财务模型 Excel
- 风险清单
- 90 天行动计划

## 二、对照表：理想 vs 当前 PoC

| 阶段 | 理想做的 | 当前 PoC | 差距 |
|---|---|---|---|
| 0 需求澄清 | 多轮对话搞清楚 6 个维度 | 一句话直接开干 | ❌ 完全缺失 |
| 1 品类宏观 | Trends + 搜索量 + GMV + 季节 + 合规 | 只 Google Trends | 🟡 严重不足 |
| 2 竞争格局 | BSR + Movers&Shakers + CR4 + 价格带空白 + 评分分布 | 抓 10-16 个搜索结果 | 🟡 太浅 |
| 3 痛点挖掘 | 真抓差评 + 多平台聚类 + 社媒 | 10 条样例 LLM 提炼 | 🟡 样本不真 |
| 4 候选筛选 | 多维评分矩阵 + 画像卡 | 跳过 | ❌ 缺失 |
| 5 利润可行性 | 14 项成本 + 三档场景 + 盈亏点 + 现金流 + 压测 | 单层公式 | 🟡 严重不足 |
| 6 差异化定义 | 改进点 + 参数表 + 定价节奏 + 故事 | 给两条差异化点 | 🟡 太轻 |
| 7 风险扫描 | 专利/商标/政策/物流/诉讼 | 完全跳过 | ❌ 缺失 |
| 8 决策输出 | SKU 列表 + 主推 + 财务模型 + 90天计划 | 一段建议 + 一个表 | 🟡 不完整 |

**关键差距：8 个阶段里 3 个完全缺失（澄清/筛选/风险），4 个深度不足。**

## 三、要把 Agent 改成"理想流程"，需要做什么

### 3.1 加新工具（function calling 工具池）
| 新工具 | 用途 | 数据源 |
|---|---|---|
| `clarify_requirements` | 反问商家补全需求 | 对话 |
| `get_amazon_bestsellers(category)` | 抓类目 Top100 BSR | 自有爬虫 |
| `get_movers_shakers(category)` | 抓 24h 上升榜 | 自有爬虫 |
| `get_product_reviews(asin, n=50)` | 真抓某款的差评 | 自有爬虫 |
| `analyze_price_distribution` | 算价格带空白 | pandas |
| `analyze_brand_concentration` | 算 CR4/CR10 集中度 | pandas |
| `full_cost_breakdown` | 14 项成本完整测算 | 公式+汇率+关税表 |
| `breakeven_analysis` | 盈亏平衡 + 现金流 | 公式 |
| `search_patents` | Google Patents 检索 | API |
| `search_trademark` | USPTO 商标检索 | API |
| `search_social_complaints` | Reddit/TikTok 吐槽 | 社媒爬虫 |

### 3.2 用 Skill 包改造系统提示
按 backend steering §7：把"选品方法论"做成一份 Markdown skill `procurement-research.md`，
Agent 第一步先 `load_skills(["procurement-research"])` 加载方法论，然后按阶段推进，避免它"凭感觉走"。

skill 内容就是上面的 8 阶段，配每阶段的工具清单。

### 3.3 多轮对话（必要时反问）
当前是单轮独白，应改成：
1. 商家：调研蓝牙耳机
2. Agent: "好的，先帮我确认几点：(1) 目标市场是 US/EU？ (2) 走 Amazon FBA 还是独立站？ (3) 单品采购预算？"
3. 商家答完，再开始正式 8 阶段调研

### 3.4 用 deepseek-v4-pro 做最终决策
Flash 用于工具编排和数据采集（快、便宜），最终决策（蓝海判断、差异化定义、风险综合）切到 v4-pro，深度推理更靠谱。

## 四、执行优先级（下一步该改什么）

按"价值/成本"排序：

1. ⭐⭐⭐ **加 BSR / Best Sellers / 评论真抓 工具** — 把"竞争格局"和"痛点挖掘"补到位，是最大短板
2. ⭐⭐⭐ **完整成本测算 + 盈亏点** — 卖家最关心的就是钱
3. ⭐⭐ **加需求澄清环节**（多轮对话） — 改 Agent 循环逻辑
4. ⭐⭐ **写 procurement-research.md skill** — 让 Agent 按方法论走
5. ⭐ **专利/商标检索**（Google Patents / USPTO 公共 API） — 风险扫描
6. ⭐ **Pro 模型用于最终决策** — 切模型即可

完成 1-4 后，第一环节才算真正"做好"。
