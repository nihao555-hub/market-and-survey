# 工具库：数据采集 / 爬虫（横切环节 ①选品 / ⑤跟卖监控 / ⑩复盘）

> 目标：用多个开源爬虫项目互为冗余，构建商业级、高鲁棒性的采集层。
> 原则：避开收费方案（不用 firecrawl 云）；优先 MIT/Apache/BSD 许可证；分层组合。
> ✅ star 与许可证已于 2026-05 核实（数据源 shields.io / badgen），下方表格为核实后数据。

---

## 关于 firecrawl 的说明
- firecrawl 开源版**可自托管、免费**，收费的是其云 API。
- 缺点：依赖较重（需自备 LLM key、Redis、Playwright 等），且为 AGPL-3.0（商用闭源有传染性风险）。
- 结论：按需求避开，用下面的免费组合替代。

---

## 采集能力分层（先理解架构，再选型）

```
第1层 HTTP 请求（最快最省，适合无 JS 渲染的接口/页面）
   └─ stealth-requests / cloudscraper / curl_cffi(TLS指纹)

第2层 解析 + 自适应（页面改版不易崩，省维护）
   └─ Scrapling / Scrapy(+parsel) / selectolax

第3层 浏览器自动化 + 反检测（应对 JS 渲染 + Cloudflare/DataDome）
   └─ nodriver / zendriver / patchright / pydoll / botasaurus

第4层 反检测浏览器内核（最强伪装，指纹级）
   └─ camoufox(Firefox系) / lightpanda

第5层 调度 / 框架 / 规模化（任务队列、代理轮换、去重、存储）
   └─ Crawlee / Scrapy / katana / colly / SeleniumBase

第6层 无代码 / 可视化（给非技术运营用，或快速搭竞品监控机器人）
   └─ Maxun

第7层 验证码 / Cloudflare 挑战突破（被挡时的兜底）
   └─ FlareSolverr / CloudflareBypassForScraping / Turnstile-Solver

第8层 代理池 / IP 轮换（规模化的命脉，反封禁）
   └─ proxy_pool / mubeng / rota / Resin / requests-ip-rotator

第9层 指纹生成 / 隐身插件（伪装更逼真）
   └─ fingerprint-suite / puppeteer-extra-stealth
```

## 推荐的「多引擎冗余」组合策略（高鲁棒性）

按「成本从低到高」自动降级，命中即止：

```
目标URL
  → [尝试1] curl_cffi / Scrapling(http模式)      # 最快，无渲染，TLS指纹伪装
  → [失败/被挡] [尝试2] Scrapling(stealth) / pydoll  # 浏览器+反检测(MIT许可证优先)
  → [仍被挡]   [尝试3] camoufox / patchright       # 指纹级伪装内核
  → [规模化]   全程挂 代理池 + 随机UA + 限速
  调度统一交给 Crawlee 或 Scrapy 管理（队列/重试/去重/存储）
  全程经 第8层代理池 轮换 IP；被验证码挡 → 第7层 FlareSolverr 兜底
  抓取结果 → markitdown/crawl4ai 清洗 → 喂 LLM 做卖点/痛点分析
```

完整 9 层降级链（命中即止，从快到强）：
```
L1 curl_cffi(TLS)  →  L2 Scrapling(自适应解析)  →  L3 pydoll/botasaurus(浏览器反检测)
   →  L4 camoufox/lightpanda(指纹级内核)  →  L7 FlareSolverr(验证码兜底)
贯穿全程：L8 代理池轮换IP + L9 指纹注入 ；L5 框架调度 ；L6 给运营可视化
```

冗余要点：
- 同一目标站点配置 2~3 个引擎，主引擎失败自动切备用，互为兜底。
- 解析层用 Scrapling 的自适应能力（55.6k star，主力），降低站点改版导致的维护成本。
- 代理池(proxy_pool/Resin) + 指纹随机化(fingerprint-suite) 是规模化的前提。
- 被 Cloudflare/验证码挡住时，FlareSolverr(14.1k) 作为独立服务兜底。
- 想用单一强力框架省事：SeleniumBase(12.8k, UC Mode) 一个就能扛过多数反爬。
- 规模化省资源时，headless 内核可考虑 lightpanda（30.7k，专为自动化设计）。

## 许可证选型红线（商用闭源 SaaS 视角）
- ✅ 可放心用（宽松）：Scrapling(BSD)、pydoll(MIT)、botasaurus(MIT)、curl_cffi(MIT)、crawl4ai(Apache)、ScrapeGraphAI(MIT)、crawlee(Apache)、katana(MIT)、colly(Apache)、patchright(Apache)、markitdown(MIT)、playwright(Apache)、FlareSolverr(MIT)、proxy_pool(MIT)、SeleniumBase(MIT)、fingerprint-suite(Apache)、mubeng(Apache)、Resin(MIT)、puppeteer-extra(MIT)
- ⚠️ 有传染性，需隔离为独立服务/进程，不混入主代码：nodriver/zendriver/lightpanda/maxun/firecrawl(AGPL)、undetected-chromedriver/requests-ip-rotator(GPL)
- 🟡 居中：camoufox(MPL-2.0，文件级 copyleft，单独引用风险可控)
- ❓ 许可证不明确，慎用：Turnstile-Solver(NOASSERTION)
- 建议：AGPL/GPL 组件一律以独立微服务（HTTP/容器）方式调用，避免与闭源主体静态/动态链接，规避传染。
  好消息：FlareSolverr、proxy_pool 等本就设计成独立服务，天然隔离。

## 风险与合规提示（重要）
- 直接爬 Amazon/Temu/TikTok Shop 等踩平台 ToS，且反爬强、易封 IP/账号。
- 规模化采集「价格/销量」这类数据，**优先走合规付费 API（如 Keepa）**，爬虫作为补充与兜底。
- 注意各项目许可证：AGPL（maxun/firecrawl）、GPL（undetected-chromedriver）对闭源商用有传染性，标⚠️的接入前务必核实 LICENSE。
- 反检测/绕过 Cloudflare 等能力仅用于公开数据采集，避免触碰登录态、个人隐私数据。
