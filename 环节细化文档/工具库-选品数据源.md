# 工具库：选品数据源（环节①市场调研/选品）

> 核心问题：销量/BSR/价格历史数据，是否必须买 Keepa？
> 结论：不必须。但要认清——**没有任何免费开源方案能直接给"真实销量"**，
> 因为亚马逊从不公开第三方销量，所有工具(Keepa/JS/H10)给的都是 BSR 反推的估算值(准确率约75-89%)。
> 既然都是估算，我们可以自建估算模型，用免费数据源喂养。

---

## 一、必须知道的时间节点（2026 影响选型）
- ⚠️ **Amazon PA-API 5.0 将于 2026-05-15 停用**，迁移到 Creators API。现在不要再对接老 PA-API。
- ⚠️ **Google Shopping Content API 将于 2026-08-18 关闭**，迁移到 Merchant API。
- 含义：依赖官方电商 API 的方案稳定性差、政策变动频繁，应把官方 API 当"锦上添花"，核心能力建在自有采集+自建模型上。

---

## 二、数据需求拆解（哪些免费能拿，哪些必须估算）

| 数据项 | 免费能拿到吗 | 来源 | 备注 |
|---|---|---|---|
| 商品基础信息(标题/图/价/评分/评论数) | ✅ 能 | 自有爬虫(见采集工具库) | 公开页面即有 |
| BSR 排名(实时) | ✅ 能 | 商品页 / Best Sellers 页 | 公开数据 |
| 热销/趋势榜(实时) | ✅ 能 | Amazon Best Sellers + Movers&Shakers | **每小时刷新，免费金矿** |
| 评论内容(挖痛点/卖点) | ✅ 能 | 自有爬虫 | 喂 LLM 分析 |
| 品类搜索热度/季节性 | ✅ 能 | Google Trends(pytrends) | 趋势而非绝对量 |
| 关键词搜索量(精确) | 🟡 半免费 | Google 自动补全/相关搜索 免费；精确量需付费 | |
| **真实销量** | ❌ 拿不到 | 谁都没有 | 只能用 BSR 估算 |
| 价格/BSR **历史曲线** | 🟡 自建或买 | 自己定时抓存 / Keepa | Keepa 优势就在历史沉淀 |

## 三、免费 / 大厂数据源清单

### 大厂官方（免费额度）
| 来源 | 提供方 | 费用 | 用途 | 备注 |
|---|---|---|---|---|
| Google Trends (via `pytrends`) | Google | 免费 | 品类热度、季节性、地域分布 | 非官方库，官方仓库已归档但可用 |
| Amazon Best Sellers 页 | Amazon | 免费(需自抓) | 各类目 Top100 热销 | 公开页面 |
| Amazon Movers & Shakers 页 | Amazon | 免费(需自抓) | 24h 内 BSR 涨幅最大(趋势预警) | **每小时刷新，比竞品早发现爆品** |
| Amazon Creators API | Amazon | 免费(联盟资质) | 商品基础信息(替代将停用的 PA-API) | 需联盟账号，数据有限 |
| Google Merchant API | Google | 免费 | 商品/购物数据(替代将停用的 Content API) | 2026 新接口 |

### 开源项目（采集脚本/估算逻辑）
| 项目 | 许可证 | 用途 | 备注 |
|---|---|---|---|
| `GeneralMills/pytrends` | Apache-2.0 | Google Trends 非官方 API | ~13k star，官方已归档，有活跃 fork |
| `m3mnoch/sales-rank-estimator` | - | BSR→销量估算的开源思路参考 | 老项目，可借鉴算法逻辑 |
| `ScaleLeap/awesome-amazon-seller` | - | 亚马逊卖家工具/资源大全 | 找数据源的索引 |
| `mominurr/Amazon-Best-Sellers-Data-Analysis` | - | 畅销榜数据分析示例 | 参考实现 |
| 采集层全套 | 见《工具库-数据采集与爬虫》 | 抓 Best Sellers / 商品页 / 评论 | Scrapling 等 |

### 免费在线工具（可参考其数据呈现，非 API）
- ecomscout.com/tools/trending-products — 结合 Movers&Shakers + Google Trends 的免费趋势榜(可参考思路)
- Amazon 官方 "Product Opportunity Explorer"(卖家后台) / 官方免费 sales estimator
- bookbeam.io 等免费 BSR→销量计算器(图书类，可参考换算逻辑)

### 付费但可选（按需，不强制）
- **Keepa**：优势是**多年价格/BSR 历史沉淀**(这点自建要从零积累时间)，API 较便宜。预算允许时作为历史数据补充。
- 卖家精灵/Helium10/JungleScout：估算成品，贵，可作校准基准(triangulation)。
- 第三方 Scraper API(Apify/Scrapingdog 等)：按量付费，省反爬维护。

## 四、推荐方案：自建"免费数据 + 自研估算模型"

```
[免费数据采集层]
  ├─ Amazon Best Sellers / Movers&Shakers  →  发现趋势爆品(每小时)
  ├─ 商品详情页(BSR/价格/评分/评论数)        →  自有爬虫定时抓
  ├─ 评论内容                                →  LLM 提炼痛点/卖点
  └─ Google Trends(pytrends)                →  品类热度/季节性
            │
            ▼
[自建历史库]  定时快照入库 → 自己积累 价格/BSR 历史曲线(替代 Keepa 的历史)
            │
            ▼
[自研 BSR→销量 估算模型]
  按类目建 "BSR区间 → 月销量区间" 映射表，用少量已知样本校准
  (可选: 初期用 1 个付费工具的数据校准自己的模型，之后甩开)
            │
            ▼
[LLM 分析层 = 差异化价值]
  利润测算 + 蓝海/红海判断 + 差异化机会点 + 选品决策建议
```

要点：
- **趋势发现完全免费**：Movers&Shakers 每小时刷新，是早于竞品发现爆品的免费金矿。
- **历史数据靠时间积累**：今天开始定时抓存，3 个月后你就有了自己的历史曲线，绕过 Keepa。
- **销量估算自己建模**：既然全行业都是估算，自建模型 + 少量校准即可，不必长期依赖 Keepa。
- **真正的护城河在 LLM 分析层**，不在数据本身。

---

## 五、结论（回答"一定要用 Keepa 吗"）
- ❌ 不一定。免费方案可覆盖：趋势发现、BSR、价格、评论、季节性。
- 🟡 唯一短板是"价格/BSR 历史曲线"——要么自己从今天起积累，要么初期买 Keepa 补历史。
- ✅ 建议：MVP 阶段**完全用免费数据源自建**，先跑通；若客户强需要"历史回溯"，再按需接 Keepa 作为可选增强，不作为强依赖。
