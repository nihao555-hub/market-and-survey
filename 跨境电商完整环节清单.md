# 跨境电商完整环节清单

> 用途：作为产品规划的主线文档。后续我们逐个环节填充「卖家痛点 → AI/工具方案 → 开源项目 → 商业化方式」。
> 生图环节默认直接用 GPT Image API，不自建。

---

## 阶段总览

```
① 市场调研/选品 → ② 供应链/采购 → ③ Listing 制作 → ④ 本地化 →
⑤ 多平台刊登/铺货 → ⑥ 营销推广 → ⑦ 客服/售后 → ⑧ 订单/物流/库存 →
⑨ 财务/对账/合规 → ⑩ 数据复盘/迭代
```

## ① 市场调研 / 选品
- [ ] 趋势洞察（品类热度、季节性、关键词搜索量）
- [ ] 竞品分析（价格、销量、评分、评论、卖点）

- [ ] 选品决策（是否上架、定价区间）

### 开源轮子调研结论（star 为量级参考，以 GitHub 实时为准）

> 核心判断：把能力分成「采集(手脚) / 数据(眼睛) / 分析(大脑) / 看板(UI)」四层。
> 采集和看板可直接用高 star 开源；销量/搜索量数据是商业软件护城河，靠付费 API 补；分析洞察自研做差异化。

| 子任务 | 能否用开源 | 推荐项目 | 备注 |
|---|---|---|---|
| 竞品采集 | ✅ 高 star 成熟 | firecrawl(~57.7k, AGPL/有云)、crawl4ai(~50k, Apache-2.0)、scrapy(~55k, BSD) | crawl4ai 许可证最适合商用 |
| 价格/销量数据 | 🟡 免费自建优先 | 见《工具库-选品数据源》。免费源(Best Sellers/Movers&Shakers/自抓) + 自研估算模型；Keepa 仅作可选历史补充 | 销量全行业都是BSR估算值，可自建 |
| 趋势/季节性 | 🟡 能但量不精确 | pytrends(~13k，官方已归档但可用/有 fork) | 品类热度、季节性 |
| 关键词建议 | 🟡 建议词能/搜索量不能 | open-seo、seo-keyword-research-tool | 精确搜索量需付费 API |
| 评论/卖点分析 | ✅ 直接用 LLM | 不用专门项目，抓取后喂 LLM | 优于老式 NLTK 情感分析 |
| 利润测算 | ➖ 不需要轮子 | 自写公式 (几十行) | 可嵌入 BI 看板 |
| 蓝海判断/差异化 | ❌ 自研(大脑) | open-deep-research(~6k) 当调研底座 | 你们的核心价值点 |
| 选品决策看板 | ✅ 高 star | Metabase(~46k, AGPL/商业版)、Superset(~71k, Apache-2.0) | 非技术卖家可拖拽建面板 |

数据源决策（详见《工具库-选品数据源.md》）：
- ❌ 不强制用 Keepa。免费源可覆盖趋势/BSR/价格/评论/季节性。
- ⚠️ Amazon PA-API 2026-05-15 停用、Google Content API 2026-08-18 关闭，不依赖官方旧 API。
- ✅ MVP 用"免费数据 + 自研 BSR→销量估算 + LLM 分析"，历史曲线靠自己定时抓存积累。
- 风险：直接爬亚马逊踩 ToS、反爬强，用采集工具库的多引擎冗余方案降低封禁。

## ④ 本地化（多语言/多市场）

- [ ] 文化适配（单位、尺码、节日、合规用语）
- [ ] 关键词的本地化（不是直译，是当地搜索习惯）
- [ ] 价格/货币/税费适配

- [ ] 批量刊登 / 批量改价 / 批量上下架
