# Skill: 跨境电商选品调研方法论

> 这是一份给选品 Agent 的工作手册。当你接到选品调研任务时，**严格按这 8 个阶段推进**，不要跳步、不要凭感觉。
> 对应工具已在工具池中注册，按阶段调用对应工具。

## 🚫 数据真实性铁律（违反将导致报告报废）

1. **候选品 = 真实获取的 ASIN**：所有候选 SKU 的 ASIN/标题/售价/评分必须来自 `get_bestsellers / search_products / get_movers_shakers / get_movers_shakers` 抓到的真实商品。
2. **提候选品前先看池子**：先调 `get_asin_pool()` 看当前已采集到哪些真 ASIN。
3. **每个候选品必须 validate**：在用 ASIN 做利润测算/IP 检查前，调 `validate_candidate(asin)` 确认在池中。
4. **采购成本必须查 1688**：用 `estimate_procurement_cost(中文关键词, target_sale_price_usd)` 拿真实区间，不允许凭印象。
5. **月销量必须用 BSR 算**：从 `get_bestsellers` 返回的 `estimated_monthly_sales` 字段取，不允许 LLM 拍。
6. **对标竞品用真售价**：对标某个竞品时，价格必须从池子里查（不要说"对标 P30i $28"如果池子里 P30i 真售价是 $69.99）。
7. **品牌候选名要 IP 检查**：所有想用的品牌名必须经过 `quick_ip_check` 校验商标可用性，不要现编。

## 阶段 0 · 需求澄清（永远从这里开始）

**触发条件**：商家给的需求里缺少以下任一关键信息时，必须反问。

必问 6 项（缺哪问哪，一次问完）：
1. 目标市场（US / EU / SEA / 全球？）
2. 目标平台（Amazon / Walmart / 独立站 / 多平台铺货？）
3. 资金预算（月度采购预算 + 单 SKU MOQ 心理阈值）
4. 物流方式（FBA / 海外仓 / 自发货？）
5. 商家定位（白牌走量 / 自有品牌精品 / 代工贴牌？）
6. 排除项（不做的品牌、专利雷区、敏感品类）

**输出**：用编号清单一次问完。商家答完才进入阶段 1。
不要在没拿到答复前调用任何获取工具。

---

## 阶段 1 · 品类宏观

调用工具：
- `get_trend(keyword, geo)` — Google Trends 12 月走势
- `get_bestsellers(category)` — 类目 Top 100，看市场天花板
- 必要时：`get_movers_shakers(category)` — 24h 上升榜

判断要点：
- 趋势上升（近3月均值 > 远3月）= 加分
- 季节性峰值在哪几个月，对应备货节奏
- 类目 Top 100 平均价 / 评分 / 评论数 = 类目难度

---

## 阶段 2 · 竞争格局

调用工具：
- `search_products(platform, keyword, limit=20)` 在多个平台抓
- `analyze_market_structure(products)` 算 价格带分布 / 评分中位 / 品牌集中度

输出指标：
- **价格带直方图**：哪个区间空白（机会区）
- **评分门槛**：Top 20 评分中位（< 4.3 风险大）
- **CR4 / CR10**：头部品牌占比（>60% 红海）
- **新品占比**：评论数 < 100 的占多少

---

## 阶段 3 · 痛点挖掘（差异化机会的来源）

调用工具：
- `get_reviews(asin, n_pages=3, star_filter='critical')` 抓真实差评
- `analyze_reviews(reviews)` LLM 提炼 Top 痛点 + 卖点 + 机会建议

操作要求：
- 至少抓 **3 个头部竞品**的差评（不少于 30 条/品）
- 痛点要带"出现频次"数据，不能只列名
- 给 1-2 条**可工程化改进**的差异化点（不要给"提升品质"这种废话）

---

## 阶段 4 · 候选品筛选

从阶段 1-3 数据里挑出 **3-5 个 SKU 候选**，每个出画像卡：
- 标题、价格、评分、评论数、估算月销
- 主要卖点、主要差评
- 蓝海评分（综合需求/竞争/差异化空间）

不要在没看完阶段 2-3 数据前提名候选品。

---

## 阶段 5 · 利润可行性（钱算清楚）

调用工具：
- `full_cost_breakdown(sale_price, procurement_cost, moq, monthly_sales_estimate)` — 14 项成本完整测算
- `stress_test(sale_price, procurement_cost)` — 压力测试

输出：
- 三档定价方案对比（低/中/高）
- 每方案的净利、毛利率、盈亏点、verdict
- 压测：广告 ACOS 翻倍 / 退货 15% / 汇率 -10% 时还能否盈利
- 资金占用：MOQ × 单价 + 头程，决定备货上限

判断红线：
- 毛利 < 10% → 否决
- 盈亏点 > 预估月销 → 风险高

---

## 阶段 6 · 差异化产品定义

输出：
- 1-2 个**核心差异化点**（来自阶段 3 痛点）
- 关键参数表（哪里超过竞品、哪里持平、哪里牺牲）
- 包装/赠品/捆绑卖点
- 品牌定位话术（一句话 + 三个关键词）
- 目标定价 + 折扣节奏（首发 30 天 / 稳定期）

---

## 阶段 7 · 风险扫描

调用工具：
- `quick_ip_check(keyword, brand_candidate)` — 专利 + 商标快速检索

必查项：
- Google Patents 关键词检索，列出最近 5 年高引专利
- 候选品牌名 USPTO 商标查询
- 平台政策风险（电池 / 液体 / 磁铁 等限品类）
- 历史诉讼重灾区（蓝牙耳机、按摩枪、加热马甲等）

输出：风险清单 + 是否需更换品牌名 / 规避特定专利。

---

## 阶段 8 · 决策输出

整合阶段 1-7 全部数据，输出：

```
## 候选品决策表
| SKU | 估算月销 | 售价 | 净利 | 毛利率 | 盈亏点 | 决策 | 理由 |
|-----|---------|------|------|--------|--------|------|------|
| ... | ... | ... | ... | ... | ... | 上架/观察/放弃 | ... |

## 主推建议（1-3 款）
- 产品定义、定价、差异化点

## 风险清单
- 专利/商标/政策/物流

## 90 天行动计划
- D0-30 备货 + 检测 + 上架
- D31-60 广告测款 + 评论积累
- D61-90 优化 listing + 扩品

## 财务摘要
- 总投入 / 月度盈亏点 / 预期 ROI
```

## 模型切换策略

- 阶段 0-4 用 `deepseek-v4-flash`（快、便宜）
- 阶段 5-8 切到 `deepseek-v4-pro`（最终决策需要深度推理）

**绝对禁止**：在阶段 0 没问清需求前直接抓数据。
**绝对禁止**：在阶段 5 没算完 14 项成本前给"建议上架"结论。
**绝对禁止**：在阶段 7 没查专利商标前给品牌定位。
